# Contributing

OpenHolidaysKit is currently in the scaffold and specification phase.

## Language

Use English for code, documentation, commit messages, pull request titles, and
pull request descriptions.

## Development Principles

- Implement against the specifications in `specs/`.
- Keep Swift and Kotlin behavior semantically aligned.
- Prefer small, reviewable changes.
- Add or update golden fixtures for behavior changes.
- Do not silently ignore unsupported holiday rules.
- Keep generated data reproducible.

## Pull Requests

Pull requests should include:

- a short problem statement
- the implementation summary
- verification steps
- data source and license notes when data changes

Data imports must include attribution updates and the upstream source reference.
